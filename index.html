2.4 GHz Wireless
Signal Scanner

Passive RF channel detection using ESP32, nRF24L01, and a 0.96-inch OLED. Sweeps all 128 channels of the 2.4 GHz band and displays active signals one by one in real time.

Operation

How It Works

Four stages, running in a continuous loop from the moment power is applied.

01

Tune Channel

Radio is set to RX mode. ESP32 programs the nRF24L01 synthesiser to the next channel (0-127).

02

Sample RPD

The RPD register is read 64 times. Each read above -64 dBm increments the hit counter for that channel.

03

Threshold Check

After scanning all 128 channels, any channel with 2 or more hits is flagged as active.

04

OLED Display

Each active channel is shown on screen for 1.8 seconds, then the device loops back to step 01.

Analysis

Advantages and Limitations

An honest breakdown of what this device does well and where its boundaries lie.

Advantages

  • Completely passive: does not transmit any signal during scanning
  • Protocol-agnostic: detects Wi-Fi, Bluetooth, Zigbee, RC, and anything else in the 2.4 GHz band
  • Very low component cost, under 5 USD in total hardware
  • Standalone operation: needs no computer, no phone, no app
  • Full 128-channel sweep takes approximately one second
  • Serial output for logging when connected over USB
  • All parameters are single-line constants at the top of the sketch
  • Open source with full documentation in plain English

Limitations

  • Detection threshold is fixed at -64 dBm by the nRF24L01 silicon
  • Cannot decode, read, or identify the content of any signal
  • Scans one channel at a time, so fast burst transmissions may be missed
  • No display of SSID, device name, or any protocol metadata
  • Cannot distinguish between signal types on the same channel
  • 0.96-inch OLED is small, not suitable for viewing from a distance
  • nRF24L01 requires a stable 3.3 V supply with a decoupling capacitor
  • No data logging or history without additional hardware or code
Roadmap

Future Scope

Planned and suggested directions for expanding this project beyond its current state.

01 / DISPLAY

Larger OLED or TFT Screen

Upgrading to a 1.3-inch SSD1306 or a 2.4-inch ILI9341 TFT would allow a full spectrum waterfall display showing all 128 channels simultaneously as a scrolling heatmap.

02 / PROTOCOL

Protocol Identification

Adding logic to classify channels: Wi-Fi channels 1, 6, 11 and 13; Bluetooth frequency hopping range; Zigbee channels. Each detected signal could show a protocol label instead of just a frequency.

03 / LOGGING

SD Card Data Logging

An SD card module on the SPI bus would allow timestamped CSV logging of every scan cycle. This enables long-duration monitoring and offline analysis in a spreadsheet.

04 / WIRELESS

Wi-Fi Web Dashboard

The ESP32 has built-in Wi-Fi. The scan results could be served as a live JSON endpoint or a browser-based spectrum chart accessible from any device on the same network.

05 / BATTERY

Battery-Powered Portable Unit

Adding a LiPo battery, a TP4056 charging module, and a power switch would make the device fully portable. The ESP32 deep sleep between scan cycles could extend battery life to several days.

06 / ALERT

Threshold Alerts

A passive buzzer or LED bar graph could be added to give audio or visual alerts when a channel exceeds a user-defined hit count, enabling hands-free monitoring in a bag or enclosure.

07 / ENCLOSURE

3D Printed Case

A custom enclosure with a window for the OLED and a recess for the nRF24L01 antenna would protect the hardware and make the device suitable for field use and demonstrations.

08 / DUAL RADIO

Dual nRF24L01 Scanning

Two nRF24L01 modules on the same SPI bus with separate CSN pins could scan in two halves of the band simultaneously, cutting the full-sweep time from one second to roughly 500 milliseconds.
